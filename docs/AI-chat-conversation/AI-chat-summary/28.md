## 2. Recommendation: Class vs. Interface Trade-offs

### 2.1 When to Use Interfaces (What You're Doing)

| Use Case | Frontend | Backend |
|----------|----------|---------|
| **Type Contracts** | ✅ Interfaces (types.ts) | ✅ Pydantic BaseModel |
| **Data Transfer** | ✅ Plain objects | ✅ JSON serialization |
| **API Boundaries** | ✅ No runtime validation (compile-time) | ✅ Runtime validation |
| **Testing** | ✅ Easy to mock | ✅ Easy to instantiate |
| **Serialization** | ✅ Native JSON support | ✅ `.model_dump()` |

**Why your approach is correct**:
- ✅ Separation of API models (schemas) and database models (entities)
- ✅ Pydantic provides validation at API boundary
- ✅ Frontend interfaces provide type safety without runtime overhead
- ✅ Clean data flow: HTTP ↔ API Schema ↔ Database Entity

## 3. Enhanced Domain Classes (Recommended)

### 3.1 Frontend: Rich Domain Model

**Current approach**: Dumb data objects + formatting in components

**Recommended**: Move formatting logic into domain classes

```typescript
// frontend/src/lib/models/SearchResult.ts
export class SearchResultModel {
    constructor(
        public dataset: Dataset,
        public score: number
    ) {
        if (score < 0 || score > 1) {
            throw new Error('Score must be between 0 and 1');
        }
    }

    // Behavior: Format relevance for display
    getRelevancePercent(): number {
        return Math.round(this.score * 100);
    }

    // Behavior: Get display representation
    getDisplayKeywords(limit: number = 4): string[] {
        return this.dataset.keywords.slice(0, limit);
    }

    getExtraKeywordCount(limit: number = 4): number {
        return Math.max(0, this.dataset.keywords.length - limit);
    }

    // Behavior: Truncate abstract for display
    getTruncatedAbstract(length: number = 200): string {
        return this.dataset.abstract.length > length
            ? this.dataset.abstract.substring(0, length) + '...'
            : this.dataset.abstract;
    }

    // Behavior: Generate CEH catalog URL
    getCatalogUrl(): string {
        return `https://catalogue.ceh.ac.uk/documents/${this.dataset.file_identifier}`;
    }

    // Behavior: Create from API response (factory pattern)
    static fromApiResponse(data: any): SearchResultModel {
        return new SearchResultModel(data.dataset, data.score);
    }
}
```

**Usage in component**:

```svelte
<script lang="ts">
    import type { SearchResult } from '$lib/api/types';
    import { SearchResultModel } from '$lib/models/SearchResult';

        result: SearchResult;

    let { result } = $props();
    let model = $derived(SearchResultModel.fromApiResponse(result));

    // Much cleaner - all display logic is encapsulated
    let relevancePercent = $derived(model.getRelevancePercent());
    let displayKeywords = $derived(model.getDisplayKeywords(4));
```

## 7. Specific Enhancements

### 7.1 Add Value Objects

**Value Objects** are immutable objects that represent a value:

```typescript
// frontend/src/lib/models/Relevance.ts
export class Relevance {
    private readonly value: number;

    constructor(score: number) {
        if (score < 0 || score > 1) {
            throw new Error('Relevance score must be 0-1');
        }
        this.value = score;
    }

    toPercent(): number {
        return Math.round(this.value * 100);
    }

    toString(): string {
        return `${this.toPercent()}%`;
    }

    isHigh(): boolean {
        return this.value >= 0.8;
    }

    isMedium(): boolean {
        return this.value >= 0.5 && this.value < 0.8;
    }

    isLow(): boolean {
        return this.value < 0.5;
    }

    getValue(): number {
        return this.value;
    }
}
```

**Usage**:

```typescript
class SearchResultModel {
    relevance: Relevance;

    constructor(dataset: Dataset, score: number) {
        this.relevance = new Relevance(score); // Throws if invalid
    }

    getRelevanceClass(): 'high' | 'medium' | 'low' {
        if (this.relevance.isHigh()) return 'high';
        if (this.relevance.isMedium()) return 'medium';
        return 'low';
    }
}
```

**Benefits**:
- ✅ Type safety (can't create invalid Relevance)
- ✅ Behavior encapsulation (formatting, validation)
- ✅ Reusable (use Relevance everywhere score appears)
