## 2. Caching Strategy by Layer

### **LAYER 1: Service Layer (Query/Embedding Cache)**

**What to cache:**
- ✅ Query embeddings (most expensive)
- ✅ Search results (optional)
- ❌ NOT real-time chat responses

**Why:**
- Embedding generation: 50-200ms per query (CPU/GPU bound)
- If user searches "climate datasets" twice, cache hit saves 150ms
- Embedding is deterministic (same query = same embedding)

**Example from your code:**
```python
# SearchService caching implementation
def _get_or_create_embedding(self, query: str) -> List[float]:
    """Get embedding from cache or generate new one."""
    
    if self.enable_caching and query in self._embedding_cache:
        self._cache_hits += 1
        logger.debug(f"Embedding cache hit: {query}")
        return self._embedding_cache[query]
    
    # Cache miss: generate new embedding
    self._cache_misses += 1
    embedding = self.embedding_service.embed(query)
    
    # Store in cache (with LRU eviction)
    self._embedding_cache[query] = embedding
    
    # Track cache performance
    hit_rate = self._cache_hits / (self._cache_hits + self._cache_misses)
    logger.info(f"Cache hit rate: {hit_rate:.2%}")
    
    return embedding
```

**Invalidation:**
- ✅ Time-based: Clear cache every 1 hour (metadata updates)
- ✅ Size-based: Keep only 1000 most recent embeddings (LRU)
- ✅ Manual: Clear on metadata update

## 3. Cache Invalidation Strategy

### **The Problem: Stale Data**

```
Scenario: Dataset metadata updates on CEH server
┌─────────────┐
│ CEH Server  │  Updates "Climate Data" title
└─────────────┘
         ↓
┌─────────────────────────┐
│ Embedding Cache         │  OLD: Contains old embedding
│ "climate data" → [...]  │  (generated from old title)
└─────────────────────────┘
         ↓
User searches "climate" → Gets old results (STALE!) ❌
```

### **The Solution: Multi-Strategy Invalidation**

```python
# Cache invalidation strategies

class CacheInvalidationStrategy:
    """Define when and how to invalidate caches."""
    
    # Strategy 1: Time-based invalidation (TTL)
    EMBEDDING_CACHE_TTL = 3600  # 1 hour
    FILE_CACHE_TTL = 2592000    # 30 days
    
    # Strategy 2: Size-based invalidation (LRU)
    MAX_EMBEDDING_CACHE_SIZE = 1000  # queries
    MAX_FILE_CACHE_SIZE = 10 * 1024 * 1024 * 1024  # 10GB
    
    # Strategy 3: Manual invalidation (event-based)
    def invalidate_on_metadata_update(self, dataset_id: int):
        """When dataset metadata updates, invalidate related caches."""
        # Clear embeddings for queries mentioning this dataset
        self._embedding_cache.clear()  # Aggressive but safe
        
        # Clear supporting documents for this dataset
        self._file_cache.pop(f"dataset_{dataset_id}", None)
    
    # Strategy 4: Conditional invalidation (checksum)
    def validate_cache_entry(self, key: str, expected_hash: str):
        """Validate cache entry matches expected content."""
        if key in self._cache:
            actual_hash = hash(self._cache[key])
            if actual_hash != expected_hash:
                # Hash mismatch = invalidate
                del self._cache[key]
                return False
        return True


# Your implementation: Time-based + Size-based
class SearchService:
    def __init__(self, ..., cache_size: int = 1000):
        self._embedding_cache = {}  # Size-based (LRU = 1000 max)
        
        # Add TTL tracking
        self._cache_timestamps = {}
    
    def _get_or_create_embedding(self, query: str):
        # Check cache TTL
        if query in self._cache_timestamps:
            age = time.time() - self._cache_timestamps[query]
            if age > 3600:  # 1 hour TTL
                del self._embedding_cache[query]
                del self._cache_timestamps[query]
        
        # Return cached or generate new
        if query in self._embedding_cache:
            return self._embedding_cache[query]
        
        # Generate and cache
        embedding = self.embedding_service.embed(query)
        self._embedding_cache[query] = embedding
        self._cache_timestamps[query] = time.time()
        return embedding
```

```typescript
    results: [],
    lastQuery: "",

});

async function search(query: string) {
    searchStore.update(s => ({ ...s, isLoading: true }));
    
    // Call backend API (backend handles caching)

        lastQuery: query,
        isLoading: false,
    }));

```

- ✅ Reactive updates (stores trigger component re-renders)
- ✅ Backend controls cache invalidation
- ✅ Browser HTTP cache (automatic)
- ✅ User sees cached results immediately

## 6. Cache Invalidation in Practice

### **Your Implementation**

```python
# src/services/search/search_service.py

class SearchService:
    def __init__(self, ..., enable_caching: bool = True):
        self.enable_caching = enable_caching
        self._embedding_cache = {}
        self._cache_hits = 0
        self._cache_misses = 0
    
    def search(self, query: str, top_k: int = 10):
        """Search with caching."""
        
        # Cache lookup
        cache_key = f"{query}:{top_k}"
        
        # Check embedding cache
        if self.enable_caching and cache_key in self._embedding_cache:
            self._cache_hits += 1
            embedding = self._embedding_cache[cache_key]
        else:
            self._cache_misses += 1
            # Generate embedding
            embedding = self.embedding_service.embed(query)
            if self.enable_caching:
                self._embedding_cache[cache_key] = embedding
        
        # Search vector store (also has internal caching)
        results = self.vector_store.search(embedding, top_k)
        
        # Hydrate results
        return self._hydrate_results(results)
    
    def clear_cache(self):
        """Clear embedding cache (called on metadata update)."""
        self._embedding_cache.clear()
        self._cache_hits = 0
        self._cache_misses = 0
        logger.info("Embedding cache cleared")
```

### **Complete Picture**

```
User Query: "climate datasets"
    ↓
┌─────────────────────────────────────────┐
│ LAYER 1: HTTP Client Cache (Browser)    │  ← Check if response cached
│ Cache-Control: max-age=300              │  (5 min browser cache)
└─────────────────────────────────────────┘
    ↓ (cache miss)
┌─────────────────────────────────────────┐
│ LAYER 2: Service Layer Cache            │  ← Check embedding cache
│ Query: "climate datasets" → embedding[] │  (1000 most recent queries)
│ Hit rate: 45%                           │
└─────────────────────────────────────────┘
    ↓ (cache miss)
┌─────────────────────────────────────────┐
│ LAYER 3: Embedding Generation           │  ← Generate embedding (200ms)
│ Model: all-MiniLM-L6-v2                 │
│ Result: [0.12, 0.34, ..., 0.56] (384)   │
└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐
│ LAYER 4: Vector Store Search            │  ← ChromaDB vector search

└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐

│ Dataset objects retrieved from SQLite   │  (~5ms × 10 = 50ms)

    ↓

    ↓

```

- Cache hit (Layer 2): 2ms
- Cache miss (Layers 3-5): 250-300ms
- Average (45% hit rate): ~175ms
