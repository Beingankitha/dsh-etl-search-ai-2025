## Prompt
> "How should we structure error handling in Svelte components? Should we create an ErrorBoundary wrapper component? Should each component (SearchResults, ChatInterface) handle its own errors, or should errors bubble up? How do we display network errors differently from validation errors to users?"

## Chat Answer

Component error handling is about finding the right balance between **isolation** (each component handles its own errors) and **centralization** (single error handler). Your application demonstrates good patterns. This document validates your approach and suggests improvements.

### 1.2 Benefits of Component-Level Handling

✅ **Encapsulation**: Each component responsible for its UI
✅ **Reusability**: Can use component in different contexts
✅ **Clear**: Error handling code near the UI

### 1.3 Trade-offs

❌ **Duplication**: Error handling repeated in multiple components
❌ **Inconsistency**: Different components might handle same error differently
❌ **Hard to test**: Must mock error state in each test

## 2. Error Boundary Component

### 2.1 What It Does

An error boundary wraps components and catches **unhandled errors** (errors not caught by component itself):

```svelte
<!-- BAD: Component crashes silently -->
<script>
    let value = data.nested.deeply.field;  // TypeError if data is null
</script>

<!-- GOOD: Error boundary catches it -->
<ErrorBoundary>
    <YourComponent />
</ErrorBoundary>
```

### 2.2 Implementing Error Boundary

```svelte
<!-- ErrorBoundary.svelte -->
<script>
    import { onError } from 'svelte';

    let error: Error | null = null;
    let errorDetails: string = '';

    // Catch errors in child components
    onError((err) => {
        error = err;
        errorDetails = err.stack || err.message;
        console.error('ErrorBoundary caught:', err);
    });

    function dismiss() {
        error = null;
    }

    function reload() {
        window.location.reload();
    }
</script>

{#if error}
    <div class="error-boundary">
        <div class="error-header">
            <h2>Something went wrong</h2>
        </div>
        
        <div class="error-content">
            <p class="error-message">{error.message}</p>
            
            {#if import.meta.env.DEV}
                <details class="error-stack">
                    <summary>Stack trace (development only)</summary>
                    <pre>{errorDetails}</pre>
                </details>
            {/if}
        </div>

        <div class="error-actions">
            <button on:click={dismiss} class="btn-secondary">
                Dismiss
            </button>
            <button on:click={reload} class="btn-primary">
                Reload Page
            </button>
        </div>
    </div>
{:else}
    <slot />
{/if}
```

## 3. Centralized Error Handler

### 3.1 Error Handler Service

```typescript
// errorHandler.ts
export interface ErrorResponse {
    type: 'network' | 'validation' | 'auth' | 'server' | 'timeout' | 'unknown';
    message: string;
    details?: Record<string, any>;
    retryable: boolean;
    action: 'retry' | 'fix_input' | 'login' | 'report';
}

export class ErrorHandler {
    static handle(error: unknown): ErrorResponse {
        // Network errors
        if (error instanceof NetworkError) {
            return {
                type: 'network',
                message: 'Network connection lost. Please check your internet and try again.',
                retryable: true,
                action: 'retry'
            };
        }

        // Validation errors
        if (error instanceof ValidationError) {
            return {
                type: 'validation',
                message: 'Invalid input. Please check your entry and try again.',
                details: error.details,
                retryable: false,
                action: 'fix_input'
            };
        }

        // Authentication errors
        if (error instanceof AuthError) {
            return {
                type: 'auth',
                message: 'Your session has expired. Please log in again.',
                retryable: false,
                action: 'login'
            };
        }

        // Server errors
        if (error instanceof ServerError) {
            return {
                type: 'server',
                message: 'Server error. Our team has been notified. Please try again later.',
                retryable: true,
                action: 'retry'
            };
        }

        // Timeout errors
        if (error instanceof TimeoutError) {
            return {
                type: 'timeout',
                message: 'Request took too long. Please try again.',
                retryable: true,
                action: 'retry'
            };
        }

        // Unknown errors
        return {
            type: 'unknown',
            message: 'An unexpected error occurred. Please try again.',
            retryable: false,
            action: 'report'
        };
    }

    static getIcon(type: ErrorResponse['type']) {
        const icons: Record<string, string> = {
            network: '🌐',
            validation: '⚠️',
            auth: '🔒',
            server: '⚙️',
            timeout: '⏱️',
            unknown: '❌'
        };
        return icons[type];
    }
}
```

### 3.2 Using Error Handler in Store

```typescript
// In searchStore.ts
executeSearch: async (query: string) => {
    store.update((state) => ({
        ...state,
        loading: true,
        error: null
    }));

    try {
        const response = await searchService.search(query, 10);
        store.update((state) => ({
            ...state,
            results: response.results || [],
            loading: false,
            error: null
        }));
    } catch (error) {
        // Use centralized error handler
        const errorResponse = ErrorHandler.handle(error);
        store.update((state) => ({
            ...state,
            error: errorResponse.message,
            loading: false,
            results: [],
            errorMetadata: {
                type: errorResponse.type,
                retryable: errorResponse.retryable,
                action: errorResponse.action
            }
        }));
    }
}
```

## 4. Error Handling Hierarchy

### 4.1 Three-Level Error Handling

```
Level 1: Component Local
├─ SearchResults displays own errors
├─ ChatInterface displays own errors
└─ SearchBar displays validation feedback

Level 2: Store/Service
├─ SearchStore catches service errors
├─ ChatStore catches API errors
└─ Translates to user-friendly messages

Level 3: Global Error Boundary
├─ Catches unhandled errors
├─ Shows emergency fallback UI
└─ Offers reload option
```

### 4.2 When to Handle at Each Level

**Component Level** (most specific):
```svelte
<!-- SearchBar.svelte -->
<script>
    let validationError = '';
    
    function onInput(e) {
        try {
            SearchQueryValidator.validate(e.target.value);
            validationError = '';
        } catch (error) {
            validationError = error.message;  // Handle here
        }
    }
</script>
```

**Store Level** (middle):
```typescript
// searchStore.ts - handles API errors
executeSearch: async (query) => {
    try {
        const results = await searchService.search(query);
        // Success - update store
    } catch (error) {
        // Handle API errors here - visible to all consumers
        store.update(state => ({ ...state, error: ... }));
    }
}
```

**Global Level** (catch-all):
```svelte
<!-- +layout.svelte - catches everything -->
<ErrorBoundary>
    <Header />
    <slot />
</ErrorBoundary>
```

## 5. Display Error Type Appropriately

### 5.1 Different Error Types, Different UI

```svelte
<!-- Validation error - show inline -->
<input 
    bind:value={query}
    on:change={validateQuery}
/>
{#if validationError}
    <span class="error-inline">{validationError}</span>
{/if}

<!-- Network error - show toast -->
{#if networkError}
    <Toast type="error" message={networkError} />
{/if}

<!-- Server error - show modal -->
{#if serverError}
    <Modal title="Server Error">
        <p>{serverError}</p>
        <button on:click={retry}>Retry</button>
    </Modal>
{/if}

<!-- Auth error - redirect -->
{#if authError}
    <svelte:window on:load={() => goto('/login')} />
{/if}
```

```typescript
        expect(state.error).toBeTruthy();
        expect(screen.getByText('Retry')).toBeInTheDocument();

    });
```
